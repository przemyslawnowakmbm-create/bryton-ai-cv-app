    4. Update `backend/app/deps.py` — add `get_tenant_db`, `require_roles`, and `Role` enum.
       IMPORTANT: READ the current file contents first. Plan 01 already added `get_current_user` and kept the existing `get_db` re-export. Do NOT overwrite those — APPEND the new dependencies below the existing ones.

       a. Add `Role` StrEnum:

       b. Add `get_tenant_db` dependency:
          - Takes `current_user: User = Depends(get_current_user)` as input
          - For Admin role: the user might not have a tenant_id (Admins are cross-tenant). In this case, if the endpoint needs tenant context, it should pass tenant_id as a path/query param. For get_tenant_db, if user.role == "admin", require a `tenant_id` query parameter or raise 400.
          - For SM/Recruiter: look up tenant_id from user_tenant_assignments (they may have multiple; use a `tenant_id` query/header parameter to select which one, or default to first if only one). For simplicity in this plan: require `X-Tenant-ID` header for SM/Recruiter when they have multiple assignments. If only one assignment, use it automatically.
          - For Customer/Candidate: use user.tenant_id directly (they belong to exactly one tenant).
          - CRITICAL: Use `app_async_session` (bryton_app role), NOT `async_session` (superuser).
          - Inside the session context:
            ```python
            await session.execute(
                text("SET LOCAL app.current_tenant = :tid"),
                {"tid": str(tenant_id)},
            )
            ```
          - SET LOCAL — scoped to current transaction only. This is THE critical line. Without LOCAL, tenant context leaks across pooled connections.
          - Yield session, commit on success, rollback on exception.

       c. Add `require_roles(*roles: Role)` factory:
          ```python
          def require_roles(*roles: Role) -> Callable:
              async def _check(current_user: User = Depends(get_current_user)) -> User:
                  if Role(current_user.role) not in roles:
                      raise HTTPException(
                          status_code=status.HTTP_403_FORBIDDEN,
                          detail=f"Requires one of: {[r.value for r in roles]}"
                      )
                  return current_user
              return _check
          ```

       d. Add `get_admin_db` convenience dependency that wraps `get_db` with an Admin role check:

          ```python
          async def get_admin_db(
              current_user: User = Depends(require_roles(Role.ADMIN)),
              db: AsyncSession = Depends(get_db),
          ) -> AsyncGenerator[AsyncSession, None]:
              yield db
          ```
          This is just for clarity — admin endpoints use superuser engine.

          ```python
          # }
          ```
