  <action>
    1. Create `frontend/src/stores/auth.ts` — Zustand store with persist middleware:

       ```typescript
       interface User {
         id: string
         email: string
         role: 'admin' | 'sm' | 'recruiter' | 'customer' | 'candidate'
         tenant_id: string | null
         display_name: string | null
       }

       interface AuthState {
         user: User | null
         accessToken: string | null
         _hasHydrated: boolean
         setAuth: (user: User, accessToken: string) => void
         clearAuth: () => void
         setHasHydrated: (val: boolean) => void
       }
       ```

       - Use `persist` middleware with `name: 'bryton-auth'` and `storage: createJSONStorage(() => sessionStorage)` (NOT localStorage — XSS protection per locked decision)
       - Add `_hasHydrated` flag and `onRehydrateStorage` callback that sets it to true when hydration completes. This is critical for TanStack Router `beforeLoad` — without it, hard refresh will redirect to login even when the user is authenticated because Zustand hasn't rehydrated yet.
       - Export `useAuthStore` hook
       - Export a `waitForHydration()` async function that returns a Promise resolving when _hasHydrated is true (use polling or Zustand subscribe)

    2. Create `frontend/src/lib/api.ts` — API fetch wrapper:

       ```typescript
       const BASE = import.meta.env.VITE_API_URL ?? 'http://localhost:8000'
       ```

       - `apiFetch<T>(path: string, init?: RequestInit): Promise<T>`:
         - Reads `accessToken` from `useAuthStore.getState()`
         - Sets `Content-Type: application/json` and `Authorization: Bearer {token}` headers
         - Sets `credentials: 'include'` on every request (sends httpOnly refresh_token cookie)
         - On 401 response:
           - Attempt refresh: `POST {BASE}/auth/refresh` with `credentials: 'include'`
           - If refresh succeeds (200): parse response, call `useAuthStore.getState().setAuth(data.user, data.access_token)`, retry the original request ONCE
           - If refresh fails: call `clearAuth()` and `window.location.href = '/login'`
           - Use a module-level `isRefreshing` boolean + Promise queue to prevent concurrent refresh calls (if multiple requests fail with 401 simultaneously, only one refresh fires)
         - On other non-OK responses: throw an Error with the response text
         - On success: return `res.json()` typed as T

       - `apiMutate<T>(path: string, method: string, body?: unknown): Promise<T>`:
         - Convenience wrapper calling apiFetch with method and JSON.stringify(body)
  </action>

               role: user?.role ?? null,
             }
