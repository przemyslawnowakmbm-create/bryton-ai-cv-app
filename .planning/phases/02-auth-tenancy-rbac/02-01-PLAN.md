       a. `get_current_user` dependency:
          - Uses `HTTPBearer(auto_error=False)` scheme
          - Extracts credentials; raises 401 if None
          - Calls `jwt.decode(token, settings.SECRET_KEY, algorithms=[settings.JWT_ALGORITHM])`
          - Catches `jwt.ExpiredSignatureError` -> 401 "Token expired"
          - Catches `jwt.InvalidTokenError` -> 401 "Invalid token"
          - Checks `payload["type"] == "access"`, raises 401 if not
          - Fetches user from DB via `get_db` (superuser session — user lookup is cross-tenant)
          - Raises 401 if user is None or not is_active or not email_verified
          - Returns the User object

       b. Keep existing `get_db` re-export unchanged (Plan 02 adds `get_tenant_db`)

    2. Create `backend/app/api/auth.py` — FastAPI APIRouter with prefix="/auth", tag="auth":

       a. `POST /auth/register`:
          - Accepts RegisterRequest body
          - Check email not already taken (query users by email via superuser session)
          - Hash password with `hash_password()` (bcrypt, NOT passlib)
          - Create User with role="candidate" (default for public registration), email_verified=False
          - Generate email token (token_type='verify_email', expires_at=now+24h)
          - Store EmailToken in DB
          - Call `send_verification_email(email, token)`
          - Return MessageResponse("Check your email to verify your account")
          - Uses `get_db` (superuser) — no auth required, no tenant context

       b. `POST /auth/register/candidate`:
          - Accepts CandidateRegisterRequest body (includes gdpr_consent)
          - Validates gdpr_consent is True (400 if not)
          - Same flow as /auth/register but sets gdpr_consent=True, gdpr_consent_at=now
          - Return MessageResponse with verification instruction

       c. `POST /auth/login`:
          - Accepts LoginRequest body
          - Rate limited: `@limiter.limit("10/minute")` per IP via slowapi
          - Also implement per-account limiting: after 5 failed attempts for same email in 15 minutes, return 429 (track in-memory dict with TTL, or use a simple DB check on failed_login_count — use in-memory dict for simplicity)
          - Look up user by email; raise 401 "Invalid credentials" if not found
          - Check email_verified; raise 403 "Email not verified" if false
          - Verify password with `verify_password()`; raise 401 "Invalid credentials" if wrong
          - Create access token (15-min) and refresh token (7-day)
          - Generate UUID as jti for refresh token
          - Store RefreshToken in DB (hash the jti with bcrypt for storage)
          - Set refresh token as httpOnly cookie: `response.set_cookie(key="refresh_token", value=refresh_jwt, httponly=True, secure=settings.COOKIE_SECURE, samesite="lax", max_age=60*60*24*7, path="/auth/refresh")`
          - IMPORTANT: Use `secure=settings.COOKIE_SECURE` (from config), NOT hardcoded `secure=False`. This allows production to set `COOKIE_SECURE=true` via env var.
          - Return TokenResponse with access_token and user brief
          - Uses `get_db` (superuser) — login is pre-tenant-context

       d. `POST /auth/refresh`:
          - Read refresh_token from cookie (`request.cookies.get("refresh_token")`)
          - Decode JWT; validate type="refresh"
          - Look up RefreshToken by jti in DB; raise 401 if not found or revoked_at is set
          - Check token not expired
          - Fetch user; check is_active and email_verified
          - Issue new access token
          - Return TokenResponse

       e. `POST /auth/logout`:
          - Requires `get_current_user` (authenticated)
          - Read refresh_token from cookie
          - If present, decode and set revoked_at=now on the DB RefreshToken
          - Delete cookie: `response.delete_cookie(key="refresh_token", path="/auth/refresh")`
          - Return MessageResponse("Logged out")

       f. `POST /auth/verify-email`:
          - Accepts VerifyEmailRequest body
          - Look up EmailToken by token where token_type='verify_email' and used_at is null and expires_at > now
          - Raise 400 "Invalid or expired token" if not found
          - Set user.email_verified = True
          - Set email_token.used_at = now
          - Return MessageResponse("Email verified")

       g. `POST /auth/forgot-password`:
          - Accepts ForgotPasswordRequest body
          - Look up user by email; if not found, STILL return success (prevent email enumeration)
          - If user exists: generate email token (token_type='reset_password', expires_at=now+1h)
          - Call `send_password_reset_email(email, token)`
          - Return MessageResponse("If the email exists, a reset link has been sent")

       h. `POST /auth/reset-password`:
          - Accepts ResetPasswordRequest body
          - Look up EmailToken by token where token_type='reset_password' and used_at is null and expires_at > now
          - Raise 400 if not found
          - Hash new password, update user.hashed_password
          - Set email_token.used_at = now
          - Revoke all existing refresh tokens for this user (set revoked_at=now)
          - Return MessageResponse("Password reset successful")

          - `app.add_middleware(SlowAPIMiddleware)`

       c. Export the `limiter` instance so auth.py can import it
