    **api/health.py** — APIRouter with tags=["health"]:
    - GET /health → returns {"status": "ok"} (liveness, always 200)
    - GET /health/ready → checks DB (SELECT 1 via async_session) + Azure Blob (if AZURE_STORAGE_CONNECTION_STRING is non-empty, use azure.storage.blob.aio.BlobServiceClient.get_service_properties(); if empty, skip with "skipped" status). Return 200 if all checked services are ok, 503 if any fail. Follow RESEARCH.md Pattern 4.

    **api/reference.py** — APIRouter prefix="/api/reference", tags=["reference"]:
    - GET /sfia-levels → returns list[SfiaLevelResponse], ordered by level ascending. Uses Depends(get_db).

    **main.py** — FastAPI app with lifespan context manager:
    - On startup: call seed_sfia_levels(session) inside async_session context. Log startup.
    - Register routers: health_router (no prefix), reference_router.
    - Add CORSMiddleware with origins from settings.
    - Do NOT start APScheduler here yet (that's Plan 03).

    **schemas/__init__.py**, **api/__init__.py**, **services/__init__.py** — Empty init files for package structure.
