    **docker-compose.yml** — 3 services following RESEARCH.md Pattern 9:

    `db` service:
    - image: postgres:16-alpine
    - environment: POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB from .env
    - ports: ["5432:5432"]
    - volumes: pgdata:/var/lib/postgresql/data, ./backend/init-db:/docker-entrypoint-initdb.d
    - healthcheck: pg_isready with 5s interval/timeout, 5 retries

    `backend` service:
    - build: { context: ./backend }
    - ports: ["8000:8000"]
    - env_file: .env
    - depends_on: db with condition service_healthy
    - command: bash -c "PYTHONPATH=/app alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000"
    - This runs migrations before starting the server. Alembic needs PYTHONPATH=/app.

    `frontend` service:
    - build: { context: ./frontend }
    - ports: ["80:80"]
    - depends_on: [backend]

    volumes: pgdata

    **backend/Dockerfile**:
    ```dockerfile
    FROM python:3.12-slim
    WORKDIR /app
    COPY requirements.txt .
    RUN pip install --no-cache-dir -r requirements.txt
    COPY . .
    ENV PYTHONPATH=/app
    EXPOSE 8000
    CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
    ```

    **frontend/Dockerfile** — Multi-stage build following RESEARCH.md Pattern 9:
    ```dockerfile
    FROM node:20-alpine AS builder
    WORKDIR /app
    COPY package*.json ./
    RUN npm install --legacy-peer-deps
    COPY . .
    RUN npm run build

    FROM nginx:alpine
    COPY --from=builder /app/dist /usr/share/nginx/html
    COPY nginx/nginx.conf /etc/nginx/conf.d/default.conf
    EXPOSE 80
    ```

    **frontend/nginx/nginx.conf** — Following RESEARCH.md Pattern 9:
    ```nginx
    server {
        listen 80;
        root /usr/share/nginx/html;
        index index.html;

        location / {
            try_files $uri $uri/ /index.html;
        }

        location /api/ {
            proxy_pass http://backend:8000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        location /health {
            proxy_pass http://backend:8000;
            proxy_set_header Host $host;
        }
    }
    ```
    Note: /health is proxied separately since the health router has no /api/ prefix. The backend serves health at /health and /health/ready directly.

    **backend/app/services/esco_sync.py** — Following RESEARCH.md Pattern 7:
    - `async def sync_esco_skills()` function
    - Uses httpx.AsyncClient with 30s timeout
    - ESCO API base: https://ec.europa.eu/esco/api
    - Fetches /resource/skill with params: language=en, type=KnowledgeSkillCompetence, offset, limit=100
    - Paginates until empty results or len(results) < limit
    - Uses `INSERT ... ON CONFLICT DO UPDATE` (pg_insert from sqlalchemy.dialects.postgresql) on uri column
    - Updates preferred_label and synced_at on conflict
    - Adds asyncio.sleep(0.1) between pages to respect rate limits
    - Wraps entire sync in try/except with logger.error on failure (never crashes the app)
    - Logs total skills upserted on completion

    **backend/app/main.py** — UPDATE the existing main.py from Plan 01:
    - Add APScheduler AsyncIOScheduler import
    - In lifespan startup: after SFIA seeding, create scheduler, add ESCO sync job as cron (day_of_week='mon', hour=2), start scheduler
    - In lifespan shutdown: scheduler.shutdown()
    - Log that ESCO sync is scheduled
    - Keep existing router registrations and CORS middleware
