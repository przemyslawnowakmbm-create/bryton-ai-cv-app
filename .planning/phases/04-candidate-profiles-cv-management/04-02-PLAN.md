    **backend/app/services/blob_service.py** — Azure Blob async upload/download helper:
    - `async def upload_blob(blob_path: str, data: bytes, content_type: str) -> str`: Upload bytes to Azure Blob. Use `azure.storage.blob.aio.BlobServiceClient` (async, never sync). Container name from `settings.AZURE_BLOB_CONTAINER`. If `settings.AZURE_STORAGE_CONNECTION_STRING` is empty, log warning and return blob_path without uploading (dev mode). Return the blob_path on success.
    - `async def download_blob(blob_path: str) -> bytes`: Download bytes from Azure Blob. Same connection string and dev-mode fallback.
    - `def validate_file_content(contents: bytes, declared_content_type: str) -> bool`: Magic-byte validation stub. Check: PDF starts with `%PDF` (bytes `b"%PDF"`), DOCX starts with `PK` zip header (bytes `b"PK\x03\x04"`). Return True if header matches declared type. This is the virus-scan stub per locked decision (full ClamAV deferred). Add `# TODO: integrate ClamAV for full virus scanning` comment.

    **backend/app/services/candidate_service.py** — Business logic:
    - `COMPLETENESS_WEIGHTS` dict matching RESEARCH.md Pattern 4 exactly (total = 100):
      ```
      full_name: 10, email: 10, phone: 5, location: 5, availability: 5,
      rate_expectation: 10, sfia_level: 10, skills_present: 15,
      experience_present: 15, clearance_present: 5, language_present: 5, visibility_set: 5
      ```
    - `def compute_completeness(candidate, has_clearance: bool, has_language: bool) -> int`: Calculate weighted percentage. Check parsed_data JSONB for skills and experience. Return integer 0-100.
    - `async def update_completeness(db: AsyncSession, candidate_id: UUID) -> int`: Query candidate + related clearances/languages existence, call compute_completeness, update candidate.profile_complete, return new value.
    - `async def get_or_create_candidate(db: AsyncSession, user: User) -> Candidate`: Lazy upsert — find candidate by user_id, if not found create one with full_name=user.display_name or user.email, return it.
    - `async def set_current_cv(db: AsyncSession, candidate_id: UUID, cv_file_id: UUID) -> None`: Two-step: UPDATE cv_files SET is_current=False WHERE candidate_id=X, then UPDATE cv_files SET is_current=True WHERE id=cv_file_id AND candidate_id=X.
    - `async def next_cv_version(db: AsyncSession, candidate_id: UUID) -> int`: SELECT COALESCE(MAX(version), 0) + 1 FROM cv_files WHERE candidate_id=X.

    **backend/app/api/candidates.py** — Full REST API router with prefix="/candidates", tags=["candidates"]:
    CRITICAL: All endpoints use `Depends(get_db)` — NOT `get_tenant_db`. Candidates are global.

    Endpoints:
    - `GET /candidates/me` — Get own profile (Candidate role). Auto-creates if absent via get_or_create_candidate. Returns CandidateResponse. Recalculates completeness before returning.
    - `PATCH /candidates/me` — Update own profile (Candidate role). Accepts CandidateUpdate. Updates fields. Recalculates completeness. Returns CandidateResponse.
    - `PATCH /candidates/me/visibility` — Update visibility preferences (Candidate role). Accepts VisibilityPreferences. Updates candidate.visibility JSONB. Recalculates completeness. Returns CandidateResponse.
    - `GET /candidates/{candidate_id}` — Get any candidate (Admin, SM, Recruiter roles). Returns CandidateResponse.
    - `GET /candidates` — List candidates (Admin, SM, Recruiter roles). Returns list of CandidateResponse. Optional query params: search (name/email), min_sfia, max_sfia. Paginated (limit/offset, default limit=50).

    CV endpoints:
    - `POST /candidates/me/cv` — Upload CV (Candidate role). Accept UploadFile. Validate: file size <= 10MB, content_type in ("application/pdf", "application/vnd.openxmlformats-officedocument.wordprocessingml.document"). Call validate_file_content for magic-byte check. Upload to Blob at path `cvs/{candidate_id}/v{version}_original.{ext}`. Create CvFile record with parse_status="pending". If this is the first CV, set is_current=True. Return CvUploadResponse (201). Do NOT call Claude here — that is Plan 03's job. Extract raw text using pdfplumber/python-docx and store in cv_file.raw_text for later parsing.
    - `GET /candidates/me/cv` — List own CV versions (Candidate role). Returns list of CvFileResponse ordered by version DESC.
    - `POST /candidates/me/cv/{cv_file_id}/set-current` — Set a CV as current (Candidate role). Calls set_current_cv.
    - `GET /candidates/{candidate_id}/cv` — List CV versions for any candidate (Admin, SM, Recruiter).

    Security clearance endpoints:
    - `POST /candidates/me/clearances` — Add clearance (Candidate role). Accepts SecurityClearanceCreate. Creates SecurityClearance record. Recalculates completeness. Returns SecurityClearanceResponse (201).
    - `GET /candidates/me/clearances` — List own clearances (Candidate role).
    - `PATCH /candidates/me/clearances/{clearance_id}` — Update clearance (Candidate role). Accepts SecurityClearanceUpdate. If expiry_date is updated to a future date, reset alert_sent_90d/60d/30d to False (clearance renewal). Recalculates completeness.
    - `DELETE /candidates/me/clearances/{clearance_id}` — Delete clearance (Candidate role). Recalculates completeness.
    - `GET /candidates/{candidate_id}/clearances` — List clearances for any candidate (Admin, SM, Recruiter).

    Language endpoints:
    - `POST /candidates/me/languages` — Add language (Candidate role). Accepts CandidateLanguageCreate. Creates CandidateLanguage record. Recalculates completeness. Returns CandidateLanguageResponse (201).
    - `GET /candidates/me/languages` — List own languages (Candidate role).
    - `PATCH /candidates/me/languages/{language_id}` — Update language (Candidate role). Accepts CandidateLanguageUpdate. Recalculates completeness.
    - `DELETE /candidates/me/languages/{language_id}` — Delete language (Candidate role). Recalculates completeness.
    - `GET /candidates/{candidate_id}/languages` — List languages for any candidate (Admin, SM, Recruiter).

    Ownership enforcement: For all `/me` endpoints, verify `candidate.user_id == current_user.id`. For clearance/language sub-resource writes, verify the clearance/language belongs to the candidate's profile. Return 403 if ownership check fails.

    For text extraction in the CV upload endpoint: use pdfplumber for PDF, python-docx for DOCX. Wrap in try/except — if extraction fails, set raw_text=None and continue (text extraction failure should not block upload). Use `io.BytesIO(contents)` since file is already in memory.
