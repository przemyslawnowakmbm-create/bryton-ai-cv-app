  <name>Task 1: CV parser service — text extraction + Claude structured parsing</name>
  <files>
    backend/app/services/cv_parser.py
  </files>
  <action>
    Create the CV parsing service with three main functions:

    **1. Text extraction functions:**

    `def extract_text_from_pdf(contents: bytes) -> str`:
    - Use `pdfplumber.open(io.BytesIO(contents))` to extract text from all pages
    - Join pages with newlines: `"\n".join(page.extract_text() or "" for page in pdf.pages)`
    - Strip excessive whitespace
    - Return the raw text string
    - Wrap in try/except, return empty string on failure with logger.warning

    `def extract_text_from_docx(contents: bytes) -> str`:
    - Use `from docx import Document as DocxDocument`
    - `DocxDocument(io.BytesIO(contents))` to parse
    - Join paragraphs: `"\n".join(para.text for para in doc.paragraphs)`
    - Strip excessive whitespace
    - Return the raw text string
    - Wrap in try/except, return empty string on failure with logger.warning

    `def extract_text(contents: bytes, content_type: str) -> str`:
    - Dispatch to extract_text_from_pdf or extract_text_from_docx based on content_type
    - Raise ValueError for unsupported content types

    **2. Claude parsing function:**

    `async def parse_cv_with_claude(raw_text: str) -> tuple[dict, list[str]]`:
    - If `not settings.ANTHROPIC_API_KEY`, raise `HTTPException(503, "AI service not configured")`
    - Create AsyncAnthropic client: `AsyncAnthropic(api_key=settings.ANTHROPIC_API_KEY)`
    - Define the system prompt:
      ```
      You are an expert CV/resume parser. Extract structured data from the provided CV text.
      For skills, list them as free text exactly as mentioned in the CV.
      For ESCO mapping, silently map each skill to the closest ESCO occupation/skill code. This mapping is for internal analytics only.
      For SFIA level, suggest a level 1-7 based on the candidate's overall experience and provide a brief justification.
      For languages, infer CEFR levels (A1-C2) from context clues (native speaker = C2, fluent = C1, working proficiency = B2, etc.).
      For clearance mentions, extract any references to security clearances verbatim.
      If a field cannot be determined from the CV, return null for that field.
      ```
    - Use a Pydantic model for Claude's output that INCLUDES skills_esco_codes (internal field):
      ```python
      class _ClaudeParseOutput(BaseModel):
          full_name: str | None = None
          email: str | None = None
          phone: str | None = None
          location: str | None = None
          availability_date: str | None = None
          rate_expectation: float | None = None
          rate_currency: str | None = None
          sfia_level_suggested: int | None = None
          sfia_level_justification: str | None = None
          skills: list[str] = []
          skills_esco_codes: list[str] = []  # internal — stored separately
          work_experience: list[dict] = []
          education: list[str] = []
          certifications: list[str] = []
          languages: list[dict] = []  # [{language, cefr_inferred}]
          clearance_mentions: list[str] = []
      ```
    - Call Claude using `messages.parse()` with try/except:
      ```python
      try:
          response = await client.messages.parse(
              model="claude-sonnet-4-6",
              max_tokens=4096,
              system=SYSTEM_PROMPT,
              messages=[{"role": "user", "content": f"Parse this CV:\n\n{raw_text}"}],
              output_format=_ClaudeParseOutput,
          )
          parsed = response.parsed_output
      except AttributeError:
          # Fallback: messages.parse() not available on this SDK version
          # Use tool_use pattern instead
          ...
      ```
    - For the tool_use fallback: define a tool with the same schema as _ClaudeParseOutput, call `client.messages.create()` with the tool, extract JSON from the tool_use content block.
    - Split the result: return a tuple of (parsed_data_dict, esco_codes_list) where parsed_data_dict contains everything EXCEPT skills_esco_codes (that field goes into esco_codes_list).
    - Wrap the entire function in try/except, log errors, re-raise as HTTPException(502, "CV parsing failed: {error}")

    **3. Orchestration function:**

    `async def parse_cv_file(cv_file_id: UUID, candidate_id: UUID, db: AsyncSession) -> None`:
    - Load the CvFile record from DB
    - If raw_text is None or empty, log warning and set parse_status="failed", parse_error="No text extracted"
    - Set parse_status="parsing"
    - Call `parse_cv_with_claude(cv_file.raw_text)`
    - On success: update the Candidate record's `parsed_data` JSONB with the parsed_data_dict, update `parsed_skills_esco` JSONB with the esco_codes_list, set cv_file.parse_status="completed"
    - Also update candidate fields from parsed data if they are currently empty: full_name, email, phone, location, availability, rate_expectation, rate_currency, sfia_level (from sfia_level_suggested)
    - On failure: set cv_file.parse_status="failed", cv_file.parse_error=str(error)
    - Commit changes
    - This function is designed to be called after upload (by the API endpoint, or by a background task in future).

    Import from app.utils.logging the logger. Import settings from app.config.
  </action>

    CV parser service is importable. Text extraction handles PDF and DOCX. Claude parsing uses AsyncAnthropic with messages.parse() and tool_use fallback. ESCO codes are separated from public parsed data. parse_cv_file orchestrates the full flow.
